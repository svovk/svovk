How not to lose sanity taking MOOC lecture notes
Thu 02 May 2019 Misc TeX / Markdown / Pandoc

This problem was a big deal for me. I like MOOCs, and I find it very valuable to make some notes during a course. Although it took time to find a way that fits my needs:

  • ability to copy-paste images
  • easy way to insert math formulas
  • output with a table of contents
  • spellchecking

I tried numerous approaches, but each of them had some downsides:

  1. Word - didn't like the way to include formulas. It took much more time to figure out how to enter a formula compared to LaTeX.
  2. LibreOffice also had a problem with formulas plus image rendering was glitchy (at least on mac)
  3. LaTeX was good for formulas although its syntax is too wordy.

I decided to fix the LaTeX approach as it looked like the most flexible one. Fortunately, it wasn't very hard to do that.

Processing

As I said writing directly in LaTeX was too verbose for me. At least for the purpose of making lecture notes. Although, I believed that LaTeX is a good option. It is an industry standard, has a very good reputation and it is definitely powerful enough to cover all my needs.

The thing is that it is obviously too powerful for that purpose and I don't need all of the LaTeX just to take notes. But there is a minimalistic format that looks like a good fit - Markdown. Stop. Why don't just convert Markdown to LaTeX? And there is a good tool for that - Pandoc. Here is how it may be used for the conversion:

pandoc --listings -s -f markdown -t latex -o output_file.tex input_file.md

Note: -s option is to generate header and footer.

Pandoc also is capable of converting directly to PDF (via tex engine)

pandoc --listings --pdf-engine=xelatex -f markdown -o output_file.pdf input_file.md

Options used:

  • --pdf-engine=xelatex is to use XeLaTeX engine which has few advantages over PdfLaTeX: may use system fonts, has better support for non-latin character encodings.
  • --listings Use the listings package for LaTeX code blocks
  • if you would like to include a custom configuration into your latex file you may use --include-in-header=listings-setup.tex

Software setup

You basically need 3 components:

  • Pandoc may be installed with homebrew: brew install pandoc
  • TeX distribution. I use MacTeX. I had to install a couple of additional packages as pandoc printed some error messages while converting to PDF:
    • tlmgr install ucharcat (the error was ! LaTeX Error: Fileucharcat.sty' not found.`)
    • tlmgr install lm-math (the error was ! The font "latinmodern-math" cannot be found.)
  • I use VimR as an editor. It has Markdown highlighting, Markdown preview, spellchecking and handy md-img-paste plugin which lets you insert images from the buffer by pressing \p (to be formal <leader>p). If you don't like VIM I am sure that there are plenty of other good editors.
