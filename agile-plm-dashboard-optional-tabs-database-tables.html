Agile PLM Dashboard Optional Tabs - database tables
Sat 19 May 2018 Agile PLM Agile PLM / Agile Database Schema

Recently I was working on a data issue which prevented a person from using Optional Tab 1 of his Dashboard. When the user clicked on the tab, Agile showed "Object does not exists" error and nothing else appeared. Researching this issue brought me to understanding of some relations in Agile DB Schema which I would like to discuss in this post.

I don't think Dashboard DB structure is something that Oracle changes frequently, but I should mention that I worked on the issue in Agile PLM 9.3.6.4 and didn't validate if all relations are the same in any other PLM versions.

Dashboard

As for many other setup entities, database schema journey starts with one of the most basic tables in Agile PLM schema - nodetable:

SELECT id, lpad('+ ',level + 1) || description node
FROM nodetable n
CONNECT BY n.parentId = PRIOR n.id
START WITH n.description = 'Dashboard Management'

This query shows us hierarchy under Dashboard Management node (Java Client) Admin -> System Settings -> Dashboard Management. Here is what it shows in my environment:

ID          NODE
----------  ----------------------
2000000001  + Dashboard Management  
...
2000000111   + Optional Tab 1
2000000112    + Table 1
2000000113    + Table 2
2000000114    + Table 3
2000000115    + Table 4
...

Now we have node ids and may query what Optional Tab 1 tables actually display:

SELECT tableId, queryId, tableName, userId
FROM query_table
WHERE tableId IN (2000000111, 2000000112, 2000000113, 2000000114)
  AND userId = <userId>

Few important points about query_table:

  • As you know, each user sets content of his/her optional tab tables. That is why query_table has userId column (it is a foreign key to agileuser table).
  • queryId is a foreign key to query table
  • tableName is a user-friendly name of the table

Note: when you define a custom dashboard table with query for all users userId will be set to NULL.

Issue

In my case, root cause of the error was following - one of the query ids specified in query_table wasn't actually defined in query table. To fix the problem, I deleted query_table row that had inconsistent data which moved the tab table to no-query-assigned state.

Please note that it is not recommended to apply any data fixes to Agile Database Schema without confirming with Oracle that they are valid.

Other tables to check

When you see an issue like that it always worth to check other tables that might have orphan records referring to the same missing row. Here are few additional tables that might refer to a query id:

  • select_list table contains information about fields to be displayed

    select * select_list where query_id = <queryId>

  • foldering table stores information about folders under which queries are stored

    select * from foldering where objectid = <queryId>

  • criteria table contains information about query filters

    select * from criteria where query_id = <queryId>
