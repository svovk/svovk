How to set up Eclipse for developing Agile PLM Script Event Handlers
Thu 03 May 2018 Agile PLM Agile PLM / Groovy / Eclipse

Groovy Script PXs were introduced in Agile 9.3.0 and since then conquered the hearts of many Agile PLM users. I find myself using Groovy Scripts very frequently as it allows deliver product faster and make your code way more expressive. In this post, I will go through steps to set up your Eclipse IDE for Groovy PX Development.

1. Install Groovy-Eclipse plugin into your Eclipse IDE

  • Visit Groovy-Eclipse GitHub wiki page: Groovy-Eclipse

  • Copy installation URL for your Eclipse version from Releases or Snapshot Builds section

  • Open Eclipse and navigate 'Help ->Install New Software...' from the top menu

  • Paste copied URL and press Enter

  • Check 'Groovy-Eclipse Feature' and compilers to be used

    Note: Most recent Agile version at the moment is 9.3.6.4 and it uses groovy-2.4.6. You may find your Groovy version by executing following from your Agile PLM home

    find . -iname groovy*

2. Create Groovy Project

  • Navigate File -> New -> Other

  • Select "Groovy Project"

  • Type project name and click Finish

  • Copy agileclasses.jar and AgileAPI.jar files to your local machine. Here is quick and easy way to find them:

    cd $AGILE_HOME
find . -iname agileclasses.jar
find . -iname AgileAPI.jar

  • Go to Project properties and add copied JARs

  • Right click on 'src' and choose new -> other

  • Choose Groovy class

  • Fill create a new Groovy class form

  • Paste Agile Groovy Event code into created class file

    import com.agile.agileDSL.ScriptObj.IBaseScriptObj

void invokeScript(IBaseScriptObj obj) {
  //script body starts here
}
