How to authenticate EBS users in JSP pages
Wed 11 July 2018 Oracle EBS Oracle EBS / JSP / Java

Almost always Oracle Applications Framework is the preferable way for creating new pages for your Oracle EBS ecosystem. But in rare cases, OAF fails to fit the needs. For example, you might need precise control over HTML code that is rendered. In such cases, Java Servlet Pages might be a good fallback.

This fallback comes at a price though - in servlet pages, you have to do a lot of things manually. This post is to share a code snippet that might be used for authenticating Oracle EBS users in a custom JSP page. Often, it is the first thing that is done.

<%@ page
 language = "java"
 import = "oracle.apps.fnd.common.WebAppsContext"
 import = "oracle.apps.fnd.common.WebRequestUtil"
%>
<%
    WebAppsContext ctx = null;
    try {
        String userName = null;
        ctx = WebRequestUtil.validateContext(request, response);
        if (ctx == null){
            throw new RuntimeException("Unable to get context");
        }
        userName = ctx.getUserName();
        %><h2>Hello, <%=userName%></h2><%
    }finally{
        if (ctx != null){
            ctx.freeWebAppsContext();
        }
    }
%>

This code is very straightforward, it gets WebAppsContext object which is then used to request current username.

WebAppsContext is a quite basic object that tracks key Oracle EBS context information like responsibility, organization and etc. Behind the scenes, all these information is obtained based on JSESSIONID cookie that is set for all pages under http://your-url/OA_HTML/ by the login screen.
